# FactFoundry.Blazor.Charts — Design Document

> **Status:** Active | **Last updated:** 2026-06-07
> **Related ADRs:** ADR-0001 (Pure SVG, No JavaScript)

## 1. Purpose

A zero-dependency charting library for Blazor applications. Charts are rendered as SVG directly from Razor components — no JavaScript, no interop, no external dependencies beyond the Blazor framework itself.

Built to power TelemetryForge's analytics dashboard while remaining a standalone open source library.

## 2. Architecture Overview

### Rendering Approach

All chart output is pure SVG generated by Razor component templates. Each chart component:

1. Accepts data and configuration via `[Parameter]` properties
2. Computes geometry (scaling, positions, paths) in C# code-behind
3. Renders SVG elements in the `.razor` template
4. Handles hover interactions via Blazor `@onmouseover`/`@onmouseout` events

No JavaScript interop is used anywhere. Tooltips are SVG `<g>` groups toggled by component state.

### Data Flow

```
User Data (ChartSeries/ChartSegment)
  → Component Parameters
    → Geometry Calculation (code-behind)
      → SVG Rendering (Razor template)
        → Interactive State (hover events)
```

## 3. Project Structure

```
src/FactFoundry.Blazor.Charts/
├── Models/
│   ├── ChartSeries.cs       — Data model for line charts (label, color, values)
│   └── ChartSegment.cs      — Data model for pie/donut charts (label, color, value)
├── Components/
│   ├── LineChart.razor(.cs)  — Time-series line chart with multi-series support
│   ├── BarChart.razor(.cs)   — Grouped/stacked bar chart (vertical or horizontal)
│   ├── DonutChart.razor(.cs) — Proportional donut chart with configurable hole
│   └── PieChart.razor(.cs)   — Proportional pie chart (solid, no hole)
└── Themes/
    └── ChartDefaults.cs      — Default color palette (12 colors)
```

## 5. Theming

Colors are resolved in priority order:
1. `Color` property on the data model (per-series/segment)
2. Default palette via `ChartDefaults.GetColor(index)` — cycles through 12 colors

The default palette uses vibrant colors that work on both light and dark backgrounds.

## 6. Leader Line Labels

When `ShowLabels` is enabled on PieChart or DonutChart, each segment gets a callout label connected by a leader line:

1. A line from the segment's outer edge at the midpoint angle to an "elbow" point (10px beyond the radius)
2. A horizontal line from the elbow outward (20px) indicating text direction
3. A small dot at the line endpoint
4. Text positioned just past the dot: "Label: Value (Pct%)"

The chart radius shrinks automatically when labels are on to make room. Hover tooltips are suppressed when labels are visible to avoid redundancy.

## 7. SVG Text Rendering

Blazor's Razor compiler reserves the `<text>` tag as a directive. SVG `<text>` elements are rendered via `MarkupString` to bypass this limitation.

## 8. Target Frameworks

Multi-targets .NET 8.0, 9.0, and 10.0 to maximize compatibility. Uses `Microsoft.AspNetCore.Components.Web` package reference (not FrameworkReference) to support both Server and WebAssembly hosting models.
